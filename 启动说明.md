# 🚀 Wget MD3 快速启动指南

## 📋 启动脚本说明

为了让用户更方便地运行项目，我们提供了三个启动脚本：

### 1. start.bat (Windows 批处理脚本)
**适用于：** Windows 用户
**特点：** 简单易用，双击即可运行
**使用方法：**
```
双击 start.bat 文件
或在命令提示符中运行：start.bat
```

### 2. start.ps1 (PowerShell 脚本)
**适用于：** Windows 用户（推荐）
**特点：** 功能强大，支持参数自定义
**使用方法：**
```powershell
# 基本运行
.\start.ps1

# 自定义端口
.\start.ps1 -Port 8080

# 跳过自动打开浏览器
.\start.ps1 -SkipBrowser

# 详细输出模式
.\start.ps1 -Verbose

# 组合参数
.\start.ps1 -Port 3000 -SkipBrowser -Verbose
```

**注意：** 如果遇到执行策略限制，请先运行：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 3. start.sh (Shell 脚本)
**适用于：** Linux/macOS 用户
**特点：** 支持彩色输出，自动检测系统环境
**使用方法：**
```bash
# 首次运行需要添加执行权限
chmod +x start.sh

# 运行脚本
./start.sh
```

## 🛠️ 常见问题解决

### 问题1：Node.js 未安装
**解决方案：**
1. 访问 https://nodejs.org/ 下载安装 Node.js
2. 选择 LTS 版本（推荐）
3. Windows 用户也可使用：`choco install nodejs`

### 问题2：端口 6868 被占用
**解决方案：**
1. 关闭占用端口的程序
2. 使用 PowerShell 脚本自定义端口：`.\start.ps1 -Port 8080`
3. 或修改项目配置文件

### 问题3：PowerShell 执行策略限制
**解决方案：**
```powershell
# 临时允许脚本执行
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

# 或永久设置（推荐）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 问题4：npm 安装依赖失败
**解决方案：**
1. 检查网络连接
2. 使用国内镜像：`npm config set registry https://registry.npmmirror.com`
3. 清除 npm 缓存：`npm cache clean --force`
